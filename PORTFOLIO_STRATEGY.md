# PORTFOLIO_STRATEGY.md

This document defines the strategy behind this portfolio website.

It explains:

• the purpose of the portfolio
• the target audience
• the positioning of the portfolio owner
• the structure of the website

AI agents and contributors should follow this strategy when modifying the project.

---

## Portfolio Purpose

This portfolio is designed to showcase a **machine learning learning journey**.

The goal is not to present the owner as an expert.

Instead the goal is to show:

* curiosity about machine learning
* ability to build ML projects
* understanding of ML workflow
* commitment to learning

The portfolio should communicate:

"A student seriously learning machine learning by building projects."

---

## Target Audience

Primary audience:

* machine learning internship recruiters
* startup founders
* developers interested in ML

Secondary audience:

* technical peers
* collaborators

The website should be easy for recruiters to scan quickly.

---

## Recruiter Psychology

Most recruiters review portfolios in **10–20 seconds**.

They typically look for:

1. projects
2. GitHub activity
3. technical skills
4. curiosity and learning

Therefore the portfolio must prioritize:

* projects
* code links
* technical explanations

---

## Portfolio Positioning

The portfolio owner is positioned as:

Machine Learning Learner
AI and Data Science Explorer
Student building ML projects

The portfolio should avoid positioning the owner as:

* ML expert
* AI researcher
* senior engineer

Authenticity is important.

---

## Portfolio Narrative

The portfolio tells a story:

1. learning programming
2. discovering machine learning
3. building ML projects
4. exploring intelligent systems

The narrative should focus on **learning progress**.

---

## Website Structure Strategy

Homepage sections are ordered strategically:

1. Hero
2. Projects
3. Skills
4. Learning Journey
5. GitHub Projects
6. Blog
7. About
8. Contact

The goal is to show **proof of work early**.

Projects appear near the top of the page.

---

## Project Strategy

Projects should demonstrate understanding of the ML workflow:

* problem definition
* dataset exploration
* data cleaning
* model training
* evaluation

Each project page should include:

Problem statement
Dataset description
EDA insights
Model used
Evaluation results

---

## Skill Strategy

Skills should reflect actual tools used in projects.

Examples:

Python
Pandas
NumPy
Scikit-Learn
Matplotlib
Seaborn
Jupyter Notebook

Skills should not be exaggerated.

---

## Blog Strategy

The blog should focus on learning and explaining concepts.

Examples:

* Understanding Logistic Regression
* Exploring the Titanic Dataset
* Lessons from Building My First ML Model

Blog posts help demonstrate thinking and understanding.

---

## GitHub Integration Strategy

GitHub is the strongest proof of technical work.

The portfolio should highlight:

* project repositories
* code links
* recent commits

Recruiters often verify projects through GitHub.

---

## Learning Journey Strategy

The timeline section shows progression.

Example stages:

* learning Python
* exploring data analysis
* building ML models
* experimenting with algorithms

This helps communicate growth mindset.

---

## Optional Feature Strategy

An interactive "Try the Model" section may be added later.

Example:

Titanic survival prediction input form.

This feature should only be added if implementation remains simple.

---

## Authenticity Principle

This portfolio must remain honest.

Avoid:

* fake achievements
* exaggerated expertise
* misleading claims

Focus on:

* learning
* experimentation
* real projects

---

## Definition of Success

The portfolio succeeds if a recruiter quickly understands:

* the student builds ML projects
* the student understands basic ML workflow
* the student is serious about learning
